## Возможности

**Happwner** извлекает чистые VPN-конфигурации и передаёт их в любые приложения. Вам <u>не нужно</u> устанавливать Happ, чтобы пользоваться всеми его подписками.

Возможности:
* **Расшифровка ссылок**. Поддерживаются форматы: `happ://crypt`, `crypt2`, `crypt3`, `crypt4`, `crypt5`. Для расшифровки <u>не требуется</u> Happ, Xposed, LSPatch/NPatch или интернет-соединение.
* **Расшифровка профилей**. Если провайдер зашифровал серверы одним из десяти ключей, встроенных в Happ, приложение автоматически расшифрует их и отобразит соответствующую надпись.
* **Подмена HWID** при запросе подписки <u>в интерфейсе Happwner</u>. Позволяет обойти лимит на число устройств и восстановить доступ к подписке при смене устройства;
* **«Мост»** — встроенная служба обновления Happ-подписок внутри любых приложений (NekoBox, Hiddify, v2rayNG, husi, Exclave, Karing и др.);
* **Обработка интент-ссылок** `happ://add` и `happ://crypt` вместо самого Happ.

Функции Xposed:
* **Перехват ссылок**. Альтернативный способ расшифровки `crypt`-ссылок. Работает во всех приложениях и может быть полезен при отсутствии ключей дешифровки (например, когда появится `crypt6`);
* **Подмена HWID <u>внутри Happ</u>** и других приложений, отмеченных как целевые в менеджере Xposed/LSPosed;
* **Разблокировка профилей `crypt`-подписок внутри Happ**. Принудительно отображает "стрелочки" возле профилей и позволяет экспортировать конфиги, даже если подписка зашифрована.
* **Быстрый вызов интерфейса Happwner** тремя одновременными касаниями, когда открыт Happ или другое целевое приложение.

**Для работы Xposed-функционала требуется установленный Xposed / EdXposed / LSPosed / Vector ИЛИ пропатченная версия Happ с помощью Xpatch / LSPatch / NPatch / FunBox.**

## Установка

1. Установите APK-файл Happwner из [Releases](https://github.com/Omegaplexx/Happwner/releases).

Приложение поддерживает **Android 5.0 (Lollipop)** и выше.

### Xposed (опционально)

#### Root
1. Установите **Happwner** и **Happ**;
2. Включите **Happwner** в менеджере **Xposed**, выберите рекомендуемые приложения;
3. Перезапустите **Happ**.

#### Без Root
1. Установите **Happwner**, **Happ** и [NPatch](https://github.com/7723mod/NPatch/releases/tag/v1.0.2) (рекомендуется версия **1.0.2**);
2. В **NPatch**: *вкладка «Управление»* → *кнопка «+»* → *«Выберите установленное приложение»* → ***Happ*** → *«Интегрированный» режим* → *«Встроить модули»* → ***Happwner*** → *«Запустить патч»*.

**NPatch не сообщает пользователю об успешной установке пропатченного приложения. Нажмите «Установить», подождите несколько секунд и проверьте Happ в списке приложений.**

## Зачем нужен Happwner?

Чтобы вернуть пользователю контроль над подписками, повысить прозрачность услуг и обезопасить себя от недобросовестных "барыг".

**Happ** — это не только VPN-клиент, но и часть коммерческой экосистемы, ориентированной на VPN-провайдеров. Happ предоставляет им расширенные возможности за счёт введения ограничений для пользователей.

* Шифрование ссылок (`happ://crypt5`) скрывает реальный URL и блокирует просмотр конфигураций серверов;
* Идентификатор оборудования (**HWID**) жёстко привязывает купленную подписку к конкретному устройству;
* Отсутствие гибких настроек в Happ не позволяет скрывать трафик от сомнительных поставщиков с помощью прокси-цепочек;
* Если в `crypt5`-ссылку встроен [ProviderID](https://www.happ.su/main/ru/dev-docs/provider-id), то раз в сутки приложение будет сравнивать хэш домена подписки, свою версию и версию ОС с теми данными, что указаны в личном кабинете продавца на [happ-proxy.com](https://happ-proxy.com);
* Продавец, добавивший **ProviderID** в зашифрованную ссылку, получает возможность [удалённо управлять приложением](https://www.happ.su/main/ru/dev-docs/app-management) без участия пользователя. Happ позволяет ему:
  * Принудительно включать отправку HWID, даже если отправка отключена в настройках;
  * Блокировать ручное изменение User-Agent, сохраняя возможность менять этот заголовок удалённо;
  * Управлять локальными SOCKS- и HTTP-прокси: перенастраивать авторизацию или отключать её;
  * Принудительно подключать пользователя к определённому серверу при запуске Happ;
  * Отключать глобальную маршрутизацию в Happ;
  * Настраивать проксирование приложений (добавлять/удалять исключения);
  * Скрывать VPN-серверы в зависимости от типа вашего подключения (Wi-Fi / моб. интернет);
  * Настраивать автоматическое тестирование серверов (автопинг) при открытии приложения;
  * Управлять типом `ping` (`via Proxy - GET/HEAD`, `TCP`, `ICMP`) и настраивать свой URL для проверки доступности серверов;
  * Менять URL подписки;
  * Задавать интервал автообновления подписки;
  * Включать автозапуск Happ при включении устройства;
  * Принудительно обновлять все подписки при каждом запуске приложения;
  * Разворачивать список серверов при обновлении подписки или полностью отключать сворачивание списка;
  * Закреплять или откреплять подписку в общем списке;
  * Изменять сортировку серверов в списке по алфавиту или пингу;
  * Управлять мультиплексированием трафика.

**Список неполный.** Параметры управления передаются через HTTP-заголовки и тело ответа при каждом обновлении подписки.
